snake get horse update log

this is not really a perfect commit history or anything.
its more just my notes of what got added while the snake game slowly got out of hand lol.

the game started as a pretty basic raylib snake project while i was learning c,
then it turned into this snake / horse / rock / roguelike thing.


basic code direction

- tried to keep the code pretty direct and c-like.
- most stuff is just structs, enums, timers, arrays, and clear game states.
- tried to keep actual game logic away from raylib drawing/input stuff where possible.
- kept deterministic xorshift rng instead of using rand.
- no png sprite spam or audio file spam. most visuals/audio are made from code.
- the file is big now, but it is still mostly straight forward systems stacked together.


fruit stuff

- started with just a basic red fruit.
- later changed fruit into primitive drawn fruit:
  - apple
  - orange
  - banana
  - cherry
- fruit can have different score values.
- still drawn with raylib shapes, not image files.


snake drawing

- changed the snake from just boring rectangles.
- added a more snake-like head, body cells, body connections, eyes, tongue, and tail.
- boosted snake has a pulsing electric/rainbow look.
- when rock shield charges are active there is a little shield drawn near the head.


title screen / options

- renamed the game title to "Snake get horse".
- added an info box on the title screen:
  - avoid falling rocks
  - collect fruit
  - horse collectible activates boost
  - boost can break rocks
- added title screen checkboxes:
  - easy mode
  - falling rocks
  - audio
- easy mode wraps around walls instead of dying.
- falling rocks can be disabled for easier play.
- audio can be muted from the title screen.


horse collectible

- added the white horse collectible.
- horse is 2x2 cells now so it can look more like an actual little horse. ( THIS HORSE WAS HARD AS FUCK TO CODE SO LEAVE ME ALONE I KNOW IT SUCKS)
- horse milestones are spaced out like:
  - 10
  - 25
  - 45
  - 70
  - 100
  - 140
  - 190
  - 260
  - then it keeps spacing out after that
- collecting a horse:
  - adds to the Horses counter
  - grows the snake by 3
  - activates boost
- boost got reduced from 30 seconds to 22 seconds because it was too easy to chain horses.(still might be)
- boost makes the snake faster and lets it smash rocks.


falling rocks

- added falling rock hazards.
- rocks have a warning/shadow phase before they actually land.
- removed 1-cell rocks because they looked too much like fruit.
- current rock sizes are:
  - 2-cell rock
  - 4-cell rock drawn as one bigger 2x2 rock
  - rare 6-cell rock drawn as one bigger rectangular rock
- rocks fall randomly, roughly every 5 to 12 seconds before upgrades change it.
- boosted snake can smash rocks.


rock medal / shield

- added a secret rock smashing medal.
- after enough boosted rock smash events, a rock medal can spawn.
- collecting it gives shield charges.
- shield starts at 3 charges.
- if the snake hits a rock without boost but has shield charges:
  - one charge gets spent
  - the rock gets cleared
  - the snake survives


background music

- added generated background music.
- no audio files.
- the sound is made in code using simple waveforms and noise.
- added baked music settings after messing around with the editor.
- later cleaned the audio loop a bit so it was not doing dumb per-sample divide work.


dev console / bg.audio

- added a backtick console.
- commands right now:
  - bg.audio
  - spawn.boss
- bg.audio opens the little music editor.
- spawn.boss is for testing the jockey boss.
- opening the console during gameplay pauses the game.
- closing it resumes only if the console was what paused it.


music editor

- bg.audio has sliders/options for:
  - bpm
  - volume
  - root frequency
  - waveform
  - bass pattern
  - lead density
  - scale
  - melody pattern
  - drums
  - noise
- it can apply, export, randomize, and mutate the sound settings.
- export writes bg_audio_settings.txt.


pause

- p pauses and unpauses.
- console also pauses when opened.
- pause stops:
  - snake movement
  - boost timer
  - rocks
  - boss movement
  - damage grace timer
- added a pause overlay.


hud

- added a 40px hud area above the grid.
- score and other text used to sit over the playfield which was annoying.
- now the hud shows:
  - score
  - hp
  - horses
  - shield
  - jockey hp
  - boost timer
- this stops fruit/rocks/collectibles from hiding under score text.


roguelike upgrades

- added upgrade choice screens.
- first upgrade happens at 15 fruit so the player sees it sooner.
- later upgrades happen every 30 fruit.
- upgrade screen gives 3 choices by default.
- a boss reward can unlock 4 choices later.
- current upgrade pool:
  - Quick Scales: snake speed +5 percent
  - Lucky Horse: fruit can reveal bonus horses
  - Stone Patience: rockfalls arrive later
  - Long Warning: rock warnings last longer
  - Horsepower: horse boost lasts longer
  - Hard Head: rock medal gives more shield charges
  - Sweet Fruit: fruit gives more score
  - Small Appetite: some fruit skips growth
  - Stable Hooves: smashing rocks while boosted adds boost time
  - Kinder Rocks: giant rocks are less common
- reward screens can now be controlled with up/down and enter, because clicking
  a reward and then moving back to the arrow keys felt bad.


jockey boss

- added the jockey boss.
- boss starts much earlier now than the first version.
- it starts around 30 fruit, then 50, 70, 100, and keeps spacing out more.
- boss has:
  - 3 hp
  - telegraph phase
  - dash phase
  - recover phase
- boss chooses a horizontal or vertical lane.
- it shows warning cells first, then dashes across.
- touching the boss normally damages the snake.
- shield can protect the snake.
- boosted snake can damage the boss.
- beating the boss opens a boss reward screen.


boss rewards

- boss reward screen gives 3 choices.
- current boss reward pool:
  - Golden Saddle: horse boost lasts longer
  - Horse Magnet: better chance for bonus horses
  - Champion Charge: rock medals hold more charges
  - Four Choices: future upgrade screens show one extra option


hp / damage

- added hp as shards.
- 5 shards = 1 full hp.
- snake starts with 1 hp.
- max hp is capped at 5 hp right now.
- rocks and jockey use damage before game over.
- added damage grace after surviving a hit.
- damage grace can be improved through stat rewards.


chests / stat rewards

- added chest collectible.
- chests can drop when boosted snake smashes bigger rocks.
- 4-cell rocks have a lower chest chance.
- 6-cell rocks have a higher chest chance.
- chest luck stat improves the drop chance.
- touching a chest opens a 2-choice stat reward screen.
- current stat pool:
  - Shed Skin: max hp +0.2 and heal +0.2
  - Sweet Tooth: fruit score +1
  - Treasure Nose: chest chance +5 percent
  - Slip Away: longer grace after damage
  - Fruit Sense: better fruit appears more often
  - Lean Body: some fruit skips tail growth


build / polish notes

- compiled exe got updated with the newer version.
- source is still one main c file for now.
- build got changed to use a more optimized compile in my local script.
- the game is still messy in places, but in a good learning-project way.
- next obvious cleanup would probably be making the file easier to navigate
  without hiding how the actual game works.
